# Niyoplan (TaskFlow AI v2.0) — Iterative Implementation Plan

> **Project**: AI-Enhanced Project Management Suite  
> **Stack**: React + Vite (frontend) · Node.js + Express (backend) · Supabase (DB/Auth) · Groq API (AI)  
> **Deployment**: Vercel & Render  

---

## 🎯 Approach: The 5-Phase Build
Due to the massive scope of combining TaskFlow AI with Jira core features, the project is divided into 5 logical phases. Each phase delivers a functional, testable slice of the application.

### Phase 1: Foundation & Core Ticketing (MVP)
*Goal: Get the core issue tracking system live with roles and basic dashboards.*
* **Setup**: Initialize React/Vite, Node/Express, and Supabase DB/Schema.
* **Auth & Roles (New)**: User registration/login. Admin, PM, Member, and Viewer roles with access controls.
* **Ticket Generation (New)**: Replace basic cards with Jira-style Tickets (Custom IDs like `NIYO-123`, Issue Types: Epic/Story/Bug, Priority, Story Points, Reporter, Assignee).
* **Global Dashboard (New)**: High-level metrics (Ticket counts, active sprints, recent activity feed).

### Phase 2: Agile Execution (Sprints & Kanban)
*Goal: Enable teams to plan and execute work visually.*
* **Sprints (New)**: Backlog view. Create time-boxed sprints. Drag tickets from backlog to a sprint. Start and complete sprints.
* **Kanban Board**: Drag-and-drop board view filtered by the Active Sprint. Custom columns (To Do, In Progress, Done).
* **Ticket Details**: Rich-text markdown descriptions, labels, checklists, and attachments.
* **Project Dashboard (New)**: Sprint health indicators, burn-down charts.

### Phase 3: AI Engine & Timeline Integration
*Goal: Add Groq AI assistance and high-level project visualization.*
* **Groq API Integration**: Secure proxy backend for AI requests.
* **AI Features**: Generate ticket descriptions, suggest ticket priority, summarize sprint progress.
* **Gantt Chart**: Interactive horizontal timeline for Epics and Tickets with drag-to-resize and finish-to-start dependencies.

### Phase 4: Daily Workflows (DSM & Fast Views)
*Goal: Improve daily team communication and task visibility.*
* **DSM (Daily Stand-Up)**: Yesterday/Today/Blocker entry forms. Team grid view.
* **AI DSM Summary**: Groq-generated summaries of week-long team updates.
* **List & My Work Views**: Flat, sortable data tables and personal assigned-ticket views.

### Phase 5: Structured Meetings & ClickUp Workspace
*Goal: Formalize PM reviews and expand beyond simple tasks.*
* **PM & HR Review Sheets**: Structured meeting notes, auto-linking action items to Tickets, auto-scheduled HR follow-ups.
* **AI Meeting Summaries**: Executive summaries generated by Groq.
* **Docs Module**: Real-time collaborative text editor for project specs.
* **Goals/OKRs**: Track higher-level objectives linked to specific Tickets.
